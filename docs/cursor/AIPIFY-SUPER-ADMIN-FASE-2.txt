FASE 2A – DRIFTSFUNDAMENT
=========================

1. INSTALLASJONER
-----------------
Mål:
  Gjøre Aipify i stand til å overvåke alle kunders installasjoner.

Oversikt (tabell):
  - Kunde
  - Domene
  - Status
  - Aktive moduler
  - Sist synkronisert

Detaljvisning (per installasjon):
  - Installerte moduler
  - API-integrasjoner
  - Webhook-status
  - Feillogger
  - Synkroniseringshistorikk

Prioritet: KRITISK

Tekniske krav:
  - RPC/API for plattformomfattende installasjonsliste (platform admin only)
  - Detaljvisning skal bruke reelle data fra installations, installation_modules,
    installation_integrations
  - Feillogger og synkroniseringshistorikk: nye tabeller eller activity_logs
    utvidelse hvis nødvendig
  - Side: /platform/installations


2. SYSTEM STATUS
----------------
Mål:
  Gi Super Admin sanntidsoversikt over hele plattformens helse.

Vis tjenester:
  - Supabase
  - Resend
  - Stripe
  - Klarna
  - Vercel
  - GitHub Actions
  - Aipify API

Statusfarger:
  - Grønn  = OK
  - Gul    = Advarsel
  - Rød    = Kritisk feil

Prioritet: KRITISK

Tekniske krav:
  - Side: /platform/system
  - Helse kan hentes fra eksisterende metrics + payment provider status
  - Vercel og GitHub Actions: placeholder eller faktisk health check (API/env)
  - Ingen overbygging – start med kortbasert oversikt, utvid senere


FASE 2B – OPERASJONER
=====================

3. SUPPORT QUEUE
----------------
Mål:
  Samle alle supporthenvendelser på ett sted.

Vis:
  - Kunde
  - Prioritet
  - Kategori
  - Status
  - Ansvarlig

AI skal:
  - Analysere saken
  - Foreslå løsning
  - Eskalere ved behov

Prioritet: HØY

Tekniske krav:
  - Utvid support_cases (category, priority, ai_escalation_reason)
  - RPC: list_platform_support_queue
  - Side: /platform/support
  - AI-analyse/løsningsforslag: UI-plassholder eller enkel visning først;
    full AI-integrasjon i senere delfase


4. VARSLINGSSENTER
------------------
Mål:
  Gi Super Admin oversikt over hendelser som krever oppfølging.

Eksempler på varsler:
  - Trial utløper snart
  - Support AI feilet
  - Faktura forfalt
  - Installasjon mistet kontakt

Prioritet: HØY

Tekniske krav:
  - Ny side eller seksjon: /platform/alerts (eller integrert i /platform)
  - Bygg på get_platform_metrics + buildAlerts-mønster
  - Kortbasert, ikke overfylt dashboard
  - Varsler skal kunne filtreres: kritisk / advarsel / info


FASE 2C – AI LEDELSE
====================

5. EXECUTIVE AI DASHBOARD
-------------------------
Mål:
  La Aipify AI gi innsikt til ledelsen.

Eksempler:
  - Trials som snart utløper
  - Moduler som ikke er aktivert
  - Abonnement med best konvertering
  - Effekten av Support AI

Prioritet: MIDDELS

Tekniske krav:
  - Utvid /platform/metrics med seksjon «Executive Insights»
  - Kortbasert design – behold eksisterende hierarki
  - Innsikt fra get_platform_metrics + usage_statistics


6. KUNDEHANDLINGER
------------------
Mål:
  La Super Admin utføre handlinger direkte fra plattformen.

Handlinger:
  - Pause abonnement
  - Forleng trial
  - Oppgrader abonnement
  - Opprett supportsak
  - Send e-post
  - Deaktiver kunde

Krav:
  - Alle handlinger skal logges (se Audit Log)

Prioritet: MIDDELS

Tekniske krav:
  - Handlingsknapper på /platform/customers og /platform/customers/[id]
  - Server actions eller /api/platform/customers/[id]/actions
  - Bekreftelsesdialog for destruktive handlinger
  - Ingen ekte betalingsintegrasjon utover eksisterende placeholder-flyt


7. AUDIT LOG
------------
Mål:
  Sikre full sporbarhet.

Loggfør:
  - Admin-handlinger
  - AI-handlinger
  - Kundeendringer
  - Kritiske systemhendelser

Prioritet: MIDDELS

Tekniske krav:
  - Utvid activity_logs eller ny platform_audit_log-tabell
  - RPC: list_platform_audit_log (filtrerbar)
  - Side: /platform/audit eller fane under System
  - Vis: tidspunkt, aktør, handling, kunde, detaljer
