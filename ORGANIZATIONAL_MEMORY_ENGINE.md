# Organizational Memory Engine (ABOS)

**Feature owner:** Customer App  
**Implementation:** [Organizational Memory Engine — Phase A.34](./ORGANIZATIONAL_MEMORY_ENGINE_PHASE_A34.md)

Knowledge explains how things should work. Memory captures how things actually unfolded.

## Mission

Help organizations remember important events, decisions, and learning experiences so future actions become wiser and more effective.

## Core philosophy

- Experience has value.
- Reflection creates wisdom.
- Memory strengthens continuity.
- Organizations should not have to relearn the same lessons repeatedly.

## Memory categories

| Category | Examples |
|----------|----------|
| **Operational** | Process improvements, incident resolutions, successful interventions, workflow adjustments |
| **Relationship** | Customer preferences, communication styles, long-term partnerships, team collaboration patterns |
| **Decision** | Major decisions, rationale, trade-offs, outcomes achieved |
| **Growth** | Milestones, challenges overcome, lessons learned, improvements implemented |

## Memory capabilities

Aipify should recall previous situations, surface relevant experiences, highlight similar events, recommend lessons learned, and preserve organizational context.

## Connections

- **Self Love (A.76 planned)** — celebrates progress, recognizes resilience, reflects on growth
- **Trust** — transparent memory: what is remembered, why relevant, who contributed, how it informs recommendations
- **Knowledge Center (A.5)** — knowledge explains; memory captures experience

## ABOS principle

> Knowledge tells us what we know. Memory reminds us who we have become.

## Vision

Aipify becomes a companion that helps organizations remember their journey — gently surfacing that challenges were faced before, and ways through were found.
